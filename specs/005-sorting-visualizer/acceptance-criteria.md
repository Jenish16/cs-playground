# 005 — Sorting Visualizer: Acceptance Criteria

- [ ] Values render as a bar chart sized to the dataset.
- [ ] Selecting an algorithm regenerates the step list for the current data.
- [ ] Play animates through steps; pause halts; step advances exactly one step.
- [ ] Reset returns to the original (pre-sort) array and step 0.
- [ ] Speed control changes the animation pace.
- [ ] Comparing, swapping, and sorted states are visually distinct.
- [ ] Randomize / size controls produce a new valid dataset within range.
- [ ] At the final step the array is fully sorted and all bars marked sorted.
- [ ] The complexity table matches the selected algorithm.
- [ ] `npm run lint` passes.
